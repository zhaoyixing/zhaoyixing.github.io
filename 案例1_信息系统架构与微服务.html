案例1：信息系统架构与微服务

Ramp Coordination 系统集成案例专项

攻克：4 种架构模型 · ISA 四层 · ESB 识图 · 微服务标准答题

案例原题回顾

📋 案例背景：Ramp Coordination 机场地面协调系统集成2026-04-20 得分：14/25

问题 1（6 分）：题目给出 4 种信息系统架构模型——单机应用、（1）含 C/S 和 B/S、（2）、企业数据交换总线，要求填空。

问题 2（9 分）：根据集成架构图，填出 ESB 上各系统名和数据流向（"框中框"约定，外框=系统名，内框=数据/信息）。

问题 3（6 分）：列举微服务架构的 3 个优点和 3 个挑战。

问题 4（4 分）：填出信息系统体系结构（ISA）的四个层次（从上到下）。

失分主因：① 4 种架构模型记错（误写"分层服务架构"和"企业服务总线"）；② ISA 战略系统层级位置写反；③ ESB 框中框系统名/信息名对调；④ 微服务挑战写"资源浪费"不踩分。

考点精讲

🔴 超高频必背
信息系统 4 种架构模型

出自希赛教材第 4 章 · 4.2.3 节"信息系统架构模式"

四种模式（按演进顺序）：

  1. 单机应用模式：所有功能在一台计算机上，无网络
  2. 客户机/服务器模式（C/S）：含 B/S（浏览器/服务器）
  3. 面向服务的架构（SOA）模式：服务为中心、松耦合
  4. 企业数据交换总线：通过总线集成异构系统

⚠️ 易错："分层服务架构""企业服务总线（ESB）"都不是这 4 种模型之一！ESB 是 SOA 的实现技术。

🔴 超高频必背
信息系统体系结构（ISA）四层

从上到下顺序必须背死

  1. 战略系统（最上层）：组织战略、决策支持
  2. 业务系统：业务流程与规则
  3. 应用系统：具体应用功能
  4. 信息基础设施（最下层）：硬件/网络/数据

记忆口诀：

战略 → 业务 → 应用 → 基础设施
（企业"想做什么 → 怎么做 → 拿什么做"）

⚠️ 易错：业务系统和应用系统位置可互换（都接受），但战略系统必须在最上层！

🔴 高频
ESB 企业服务总线

ESB（Enterprise Service Bus）是 SOA 架构的核心组件，用于在异构系统之间提供消息中转、协议转换、路由。

核心特点：

  • 松耦合：消费者和服务提供者通过总线解耦
  • 协议适配：支持 SOAP/REST/JMS 等多种协议
  • 消息路由：根据规则把消息路由到目标系统
  • 数据转换：异构系统间的数据格式转换

架构图识图技巧（"框中框"约定）：

外框 = 系统名（如"乘务人员管理系统"）
内框 = 该系统提供/消费的数据（如"乘务人员信息"）
解题第一步：先找一个已给出双框的样本作基准！

🔴 超高频必背
微服务架构 - 6 大优点

案例题列举至少 3 个，须用采分关键词表述：

  1. 小服务，每个专注做一件事
  2. 化整为零，便于小团队独立开发
  3. 独立性强（独立开发/测试/部署/运行）
  4. 支持异构（每个服务可选不同语言/数据库）
  5. 容错能力强（故障隔离在单个服务内）
  6. 松耦合，易扩展、易伸缩
🔴 超高频必背
微服务架构 - 3 大挑战

案例题列举 3 个，必须涵盖以下三点：

  1. 分布式环境下的数据一致性 ⭐
    最常漏答！跨服务的事务和数据同步是难点
  2. 测试的复杂性
    服务间依赖、契约测试、端到端测试都比单体更难
  3. 运维的复杂性
    监控、链路追踪、CI/CD 流水线、容器编排都需投入

🚫 踩雷词："资源浪费风险""可用性保障挑战""服务集成测试难度大" 这些不是标准采分点，写了不给分！

🟡 高频
微服务 vs SOA 对比
  • 服务粒度：微服务更细，单一职责；SOA 服务粒度较粗
  • 通信方式：微服务多用轻量级 REST/gRPC；SOA 多用 SOAP/ESB
  • 数据管理：微服务"每服务一库"；SOA 共享数据存储
  • 部署：微服务独立部署；SOA 通常一起部署
  • 关系：微服务可看作 "细粒度的 SOA"

标准答题词 vs 踩雷词

考点 ✅ 标准答题词（踩分） ❌ 踩雷词（不给分）
4 种架构模型 单机应用 / 客户机-服务器（C/S） / 面向服务的架构（SOA） / 企业数据交换总线 分层服务架构、企业服务总线（ESB）、微服务架构、MVC
ISA 四层（顶到底） 战略系统 → 业务系统 → 应用系统 → 信息基础设施 应用系统在最上层、战略系统在底层
ESB 全称 企业服务总线（Enterprise Service Bus） 只写"服务总线"、"消息总线"
SOA 全称 面向服务的架构（Service-Oriented Architecture） 面向对象架构、服务化架构
微服务挑战 ① 数据一致性 ② 测试复杂性 ③ 运维复杂性 资源浪费、可用性挑战、服务集成测试难
C/S 模式书写 客户机/服务器模式（C/S，含 B/S） 只写 "C/S"、"客户端服务端"

架构模式对比

模式 典型代表 耦合度 适用场景
单机应用本地办公软件—无网络、单用户场景
C/S（客户机/服务器）QQ、银行客户端较紧需要丰富 UI、本地计算
B/S（浏览器/服务器）淘宝、知乎松跨平台、易部署
SOA企业 ERP 集成松（通过 ESB）异构系统集成、企业级应用
企业数据交换总线政府/银行数据交换平台松跨组织数据交换
微服务（细粒度 SOA）Netflix、阿里中台极松互联网高并发、独立演进

记忆口诀

4 种架构模型演进
单机 → C/S（含 B/S）→ SOA → 企业数据总线（从单一到分布式、再到服务化）
ISA 四层顺序
战略 → 业务 → 应用 → 基础设施（"想做什么 → 怎么做 → 拿什么做"）
微服务优点 6 字诀
小、独、异、容、松、扩（小服务 / 独立性 / 异构 / 容错 / 松耦合 / 易扩展）
微服务挑战 3 大复杂
数据一致性 + 测试复杂 + 运维复杂（"一致+两复杂"）
ESB 识图三步法
① 找已知双框 → ② 确认外=系统、内=信息 → ③ 套用约定填其他空
易错警示
ESB 不是独立架构模式，而是 SOA 的实现技术；微服务≠SOA，是细粒度的 SOA

测试题（10题，自动判分）

Q1：希赛教材中"信息系统 4 种架构模式"不包含下列哪一项？
A单机应用模式
B客户机/服务器（C/S）模式
C分层服务架构
D企业数据交换总线
✓ 正确答案：C
"分层服务架构"是自造词，4 种正确模型为：单机/C-S（含B/S）/SOA/企业数据交换总线。
Q2：信息系统体系结构（ISA）从顶到底的四层正确顺序是？
A应用系统 → 业务系统 → 战略系统 → 信息基础设施
B战略系统 → 业务系统 → 应用系统 → 信息基础设施
C信息基础设施 → 应用系统 → 业务系统 → 战略系统
D业务系统 → 战略系统 → 应用系统 → 信息基础设施
✓ 正确答案：B
战略在最上层（"想做什么"），基础设施在最下层（"拿什么做"）。业务系统和应用系统位置可互换，但战略和基础设施的位置必须固定。
Q3：以下关于 ESB（企业服务总线）的描述，不正确的是？
AESB 是 SOA 架构的核心组件之一
BESB 提供消息路由、协议转换、数据转换功能
CESB 是信息系统 4 种架构模式之一
DESB 实现服务消费者与提供者的解耦
✓ 正确答案：C
ESB 是 SOA 的实现技术，不是独立的架构模式。4 种模式中第 4 种叫"企业数据交换总线"，与 ESB 不同。
Q4：以下哪一项不属于微服务架构的 6 大优点？
A独立开发、测试、部署
B每个服务可使用不同的语言和数据库（异构）
C数据强一致性
D故障隔离在单服务内（容错强）
✓ 正确答案：C
数据一致性恰恰是微服务的"挑战"而非"优点"。微服务在分布式环境下数据一致性难以保证，需用 BASE/Saga 等模式。
Q5：案例题问微服务 3 大挑战，下列哪个表述是标准采分点？
A资源浪费风险
B服务可用性保障挑战
C分布式环境下的数据一致性
D服务粒度过细难以拆分
✓ 正确答案：C
三大标准挑战：① 数据一致性 ② 测试复杂性 ③ 运维复杂性。其余表述虽相关但不是教材标准采分点。
Q6：在 ESB 集成架构图中，外框标注"乘务人员管理系统"，内框标注"乘务人员信息"，此约定表示？
A外框=信息内容，内框=系统名
B外框=系统名，内框=该系统提供/消费的数据
C外框=数据库表名，内框=字段名
D外框和内框等价，可互换
✓ 正确答案：B
案例题"框中框"约定：外框=系统名，内框=该系统提供或接收的数据。读题第一步先找已给出的双框样本确定约定。
Q7：下列关于 SOA 与微服务的关系，正确的是？
ASOA 是微服务的子集
B微服务是细粒度的 SOA
CSOA 和微服务完全等价
D微服务是面向对象架构的别名
✓ 正确答案：B
微服务可视为细粒度的 SOA：单一职责、轻量级通信（REST/gRPC）、每服务一库、独立部署，比传统 SOA 更彻底。
Q8：B/S 架构本质上属于哪种信息系统架构模式？
A单机应用模式
B客户机/服务器（C/S）模式
C面向服务的架构（SOA）
D企业数据交换总线
✓ 正确答案：B
B/S 是 C/S 的特殊形式（浏览器作为客户端），希赛教材将其归入"客户机/服务器模式"大类。
Q9：某企业有 ERP、CRM、HR 等多个异构系统需要集成，最适合采用的架构模式是？
A单机应用模式
BB/S 模式
C面向服务的架构（SOA）/ ESB
D客户机/服务器（C/S）模式
✓ 正确答案：C
异构系统集成的标准方案是 SOA + ESB：通过总线进行协议转换、消息路由、数据格式转换，实现服务之间的松耦合集成。
Q10：以下哪一组属于微服务架构挑战的"采分关键词"？
A服务粒度过粗 + 难以拆分 + 通信延迟
B数据一致性 + 测试复杂性 + 运维复杂性
C资源浪费 + 部署慢 + 扩展性差
D单点故障 + 集中管理 + 紧耦合
✓ 正确答案：B
教材标准三大挑战："数据一致性 + 测试复杂性 + 运维复杂性"。其他描述要么相反（资源浪费/扩展性差），要么是单体架构特征。
