我的开篇flag

发表于 2017-12-31 | 分类于 Testing |

2017年的年终，趁着加班时候的安静时间，自己好好享受了一把整个办公区都属于自己的感觉。刚刚工作的时候，落到自己身上的任务单一，不要做太多决策的任务，自己只需要在角落安静的做好自己的事情就可以了。那段时间在现在看来是最美好的，所以想现在自己很希望能够找到这样的安静的时间，来做自己想做的事情。

这两天自己是高效的,自己做了三件大事:

  • 重温了一遍《肖申克的救赎》
  • 把自己一直纠结的mysql的upadte并发问题学习、探究了一遍
  • 回顾自己这一年


《肖申克的救赎》再观看

对于电影或者书籍的看法，不同人在不同时间点看，关注的重点自然不同。我更喜欢描绘小人物生活的电影，而不是superman的电影。
上次看的时候是在大三开学的时候，那个时候看完这部片的感受是：是金子总会发光的，andy在那样的环境里都可以逃脱，并且能够重获自由。
这次受到朋友圈朋友的影响，又重温了一遍，这次看的更加仔细了，对人物的对话和心理更加重视。其中几点是这次我感触比较深的

  • 1.主人公andy，在监狱经历了被欺凌、希望获得自由、自己努力感受自由、希望被浇灭、绝地重生的过程，最终凭借自己的头脑和毅力从下水道爬出来了
  • 2.负责书籍管理的老囚犯，布鲁克斯，在被放出时候的内心的恐惧。在监狱了呆了40-50年之久的人，已经慢慢熟悉了体制内的生活(监狱)，在这里他有地位、有自己的生活，一旦把他放回到自由的环境中，他无法跟的上节奏，也会失去在监狱里的地位与荣耀。对于大部分人来说，不会跳出当前的体制去想外面的事情，即使这个体制是监狱，大家还是会选择慢慢适应他，这是一个危险的信号。

mysql update的并发问题

库存管理的概念是电商业务中的一个比较重要的问题，在我们的业务中也遇到了类似的需求，需要给用户发奖品，发奖品就要控制在并发状态下的库存不会超发。我们的业务情景比较简单，只需要mysql就可以了，复杂场景可以开篇讲，这里只是点一下。

问题由证明库存扣减语句的安全性展开，自己对事务隔离级别、innodb mvcc、redo/undo log，update语句的执行过程、锁这些重新复习了一遍，最后证明这句不会存在安全性上的问题，具体证明过程后面再记录

反思与flag

上面两件事情做完之后，回想一下自己的整个2017年。自己的成长点在哪里，2018年自己需要做什么.
2017年自己的变化点主要在几个方面

  • 自己在技术上的成长点在于对业务性能优化、运维知识的理解，对于编码这项任务的理解从代码块到整体把控的方向发展
  • 开始关注除了技术之外的部分，开始关注技术之外的东西。从吴军的《硅谷来信》、张辉的微信朋友圈、一些经济书籍等其他书籍上开始了解互联网中除了技术之外的运营、产品知识及经济、商业知识。
  • 身体状况开始下降

18年希望自己能够有一些改变和突进，包括工作和生活

  • 工作上，做有意义的事情，不要把自己限制在体制内，良性发展
  • 技术学习上，经典书籍学习，不跟风。开始把自己的学习成果用git记录，与大家交流
  • 平衡生活与工作、学习，锻炼好身体

之前的博客都没有能够坚持下来，希望这次可以坚持下来！
